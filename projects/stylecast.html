How the Agent Works

7 fully automated steps. Give it a city — it handles everything else.

AUTO
1

Discover Stores

Queries OpenStreetMap or Yelp for clothing stores near any city. Found 56 stores in Los Angeles in under 10 seconds.

OSM / Yelp / Google Places
AUTO
2

Score & Qualify

Scores every store 0–100 on website quality, contact info, address completeness. Drops chains and incomplete leads. 20/56 passed.

Custom scoring engine
AUTO
3

Scrape Their Products

Visits the store's actual website and pulls clothing images using BeautifulSoup. No API needed — works on any website.

BeautifulSoup scraper
AUTO
4

AI Picks Best Garment

Sends all images to Groq Vision AI. Scores each on color, silhouette, uniqueness, and cinematic impact. Picks winner with a written reason.

Groq Llama 4 Scout Vision
AUTO
5

Generate Video Reel

Optionally places the garment on a model (virtual try-on), then animates it into a 5-second cinematic video. Falls back gracefully if any API fails.

WaveSpeedAI WAN 2.7 + Segmind
AUTO
6

Design Postcard

Creates a 4×6" print-ready postcard at 300 DPI. Front: garment with store-specific headline. Back: personalised letter + QR code linking to the video.

Pillow + qrcode (free)
AUTO
7

Physically Mail It

Sends postcard via Lob API. Lob prints, addresses, stamps, and delivers it to the store. Returns a tracking ID. Only fires after video is confirmed generated.

Lob API

How You Run It

Three commands to get started:

# 1. Install dependencies
pip install -r requirements.txt

# 2. Add your API keys to .env (4 free keys, takes 10 minutes)
WAVESPEED_API_KEY=your_key
GROQ_API_KEY=your_key
LOB_API_KEY=test_your_key
SEGMIND_API_KEY=your_key

# 3. Run on any US city
python -m pipeline.run --city "New York" --source osm --limit 10 --render-method wavespeed

# Optional: skip specific stores you've already contacted
python -m pipeline.run --city "Chicago" --source osm --limit 5 --exclude "H&M" "Zara"

What We Tried That Didn't Work

Fifteen real errors hit during development. Every one diagnosed and resolved or routed around.

What failedRoot causeHow it was fixed
HuggingFace Inference API api-inference.huggingface.co route-blocked by Indian ISP. DNS change to 8.8.8.8 had no effect — not a DNS issue. Switched to WaveSpeedAI (globally routable)
fal.ai try-on + video storage.fal.run and fal.run DNS-blocked in India. VPN gave 403 (VPN IP blacklisted by fal.ai). Abandoned. WaveSpeedAI used instead.
Replicate IDM-VTON + SVD Free trial credits exhausted. 402 on every call. Segmind + WaveSpeedAI as free alternative
Segmind IDM-VTON 406 Base64 of 800px images was ~2MB — server rejected payload size. Resize to 768px with Pillow before encoding (~200KB). Still investigating.
WaveSpeedAI — "480p" API only accepts "720p" or "1080p". Clear 400 error message. Changed resolution to "720p". Fixed immediately.
WaveSpeedAI — connection aborted Sending raw PNG as ~2MB base64 body caused server to close connection mid-upload. Upload to tmpfiles.org first, pass HTTPS URL to WaveSpeed.
WaveSpeedAI — HTTP image URL Garment URL was http:// — WaveSpeed requires HTTPS. Download and re-upload to tmpfiles.org to get HTTPS URL.
Lob — size "6x4" Lob only accepts "4x6". 422 error. Fixed string to "4x6".
Lob — missing use_type Required field not in payload. 422 "mail_use_type_can_not_be_null". Added use_type: "marketing" to request body.
Lob — wrong address key Code read payload["to"]["city"] but key was "address_city". Sent empty city. Fixed to read from address_parts directly.
OSM — 0 qualified stores All OSM stores have rating=0. Threshold was 50. Nothing passed. Lower threshold to 15 when no stores have ratings.
Groq — deprecated model llama-3.2-90b-vision-preview removed by Groq with no warning. Switched to llama-4-scout-17b-16e-instruct.
Overpass API 504 Main OSM server overloaded. Timeout on every call. Added 2 mirror URLs with automatic fallback retry loop.
Mailing without video Pipeline mailed postcards even when video failed. QR pointed to fake URL. Gates Steps 6+7 on video being confirmed generated.
OSM address parsing OSM returns "106 East 17th St Los Angeles CA 90015" (no commas). Parser split wrong. Added second parser branch for space-only format with street type detection.
